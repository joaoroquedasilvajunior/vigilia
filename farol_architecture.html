Modelo de Dados

O modelo segmenta dados em três domínios: Atores (legisladores, partidos, financiadores), Atos (proposições, votações, votos individuais), e Análise (tags temáticas, scores constitucionais, clustering comportamental). Esta separação permite atualização incremental e rastreabilidade por camada.

Decisão de design: partidos são tratados como metadados do legislador em um momento no tempo, não como entidade organizadora central. Isso reflete a realidade brasileira de migração partidária frequente e permite que o sistema agrupe deputados por comportamento real, não por legenda.

Camada de Integração de APIs

O Brasil tem um ecossistema de dados abertos razoavelmente robusto. O desafio não é acesso — é normalização, confiabilidade e frequência de atualização. A arquitetura proposta usa um padrão de ingestion pipeline com checksum-based delta sync, de modo que apenas dados novos ou modificados são processados.

// Cliente com retry exponencial e rate-limit awareness
class CamaraClient {
  private rateLimiter = new RateLimiter({ rpm: 30 }); // conservador

  async fetchPaginated<T>(
    endpoint: string,
    params: Record<string, string> = {}
  ): Promise<T[]> {
    const results: T[] = [];
    let pagina = 1;
    const itensPorPagina = 100; // máximo da API

    while (true) {
      await this.rateLimiter.wait();

      const url = new URL(`${CAMARA_BASE}${endpoint}`);
      url.searchParams.set('pagina', String(pagina));
      url.searchParams.set('itens', String(itensPorPagina));
      Object.entries(params).forEach(([k, v]) =>
        url.searchParams.set(k, v)
      );

      const res = await fetchWithRetry(url.toString(), {
        maxRetries: 3,
        backoff: 'exponential'
      });

      const json = await res.json();
      if (!json.dados?.length) break;

      results.push(...json.dados);
      if (json.dados.length < itensPorPagina) break; // última página
      pagina++;
    }
    return results;
  }

  // Busca votos de uma votação específica com deputado expandido
  async getVotosDeVotacao(votacaoId: string) {
    return this.fetchPaginated(
      `/votacoes/${votacaoId}/votos`
    );
  }

  // Busca proposições com filtro de data para delta sync
  async getProposicoesDesde(dataInicio: Date) {
    return this.fetchPaginated('/proposicoes', {
      dataInicio: dataInicio.toISOString().slice(0, 10),
      ordem: 'ASC',
      ordenarPor: 'id'
    });
  }
}

// Pipeline de ingestão com detecção de mudanças
async function syncVotacoesDelta(db: Database) {
  const ultimaSync = await db.query(
    'SELECT MAX(data) FROM votacoes'
  );
  const desde = ultimaSync.rows[0].max ?? new Date('2023-01-01');

  const client = new CamaraClient();
  const votacoes = await client.fetchPaginated('/votacoes', {
    dataInicio: desde.toISOString().slice(0, 10)
  });

  for (const votacao of votacoes) {
    const votos = await client.getVotosDeVotacao(votacao.id);

    await db.transaction(async (tx) => {
      await tx.upsert('votacoes', votacao, { onConflict: 'id_camara_api' });
      await tx.batchInsert('votos_individuais', votos.map(normalizeVoto));
    });

    // Enqueue análise constitucional se proposição nova
    if (votacao.proposicaoObjeto) {
      await queue.enqueue('score-constitucional', {
        proposicaoId: votacao.proposicaoObjeto
      });
    }
  }
}

Normalização de Setores Econômicos (TSE → Farol)

TypeScript — mapeamento CNAE → setor de influência
// Os dados do TSE usam CNAE; precisamos mapear para setores
// politicamente relevantes de forma legível ao público

const SETOR_MAP: Record<string, string> = {
  // Agronegócio
  '01': 'agronegocio',  // Agricultura
  '02': 'agronegocio',  // Silvicultura
  '03': 'agronegocio',  // Pesca
  '10': 'agronegocio',  // Alimentos e bebidas

  // Infraestrutura e construção
  '41': 'construcao',
  '42': 'infraestrutura',

  // Sistema financeiro
  '64': 'financeiro',
  '65': 'financeiro',
  '66': 'financeiro',

  // Saúde privada
  '86': 'saude_privada',
  '87': 'saude_privada',

  // Entidades religiosas
  '94': 'religioso',    // Atividades de organizações associativas
  '8412': 'religioso',  // Regulação das atividades sociais
};

function normalizarSetor(cnae: string): string {
  // Tenta match por 4 dígitos, depois 2 dígitos
  return SETOR_MAP[cnae.slice(0, 4)]
      ?? SETOR_MAP[cnae.slice(0, 2)]
      ?? 'outros';
}

// Query de correlação: financiamento × votos por setor × tema
const CORRELACAO_QUERY = `
  SELECT
    l.nome_completo,
    l.partido_atual,
    f.setor_normalizado AS setor_financiador,
    SUM(f.valor) AS total_recebido,
    COUNT(vi.id) AS total_votos_tema,
    COUNT(vi.id) FILTER (WHERE vi.voto = 'Sim') AS votos_favoraveis,
    ROUND(
      COUNT(vi.id) FILTER (WHERE vi.voto = 'Sim')::numeric
      / NULLIF(COUNT(vi.id), 0) * 100, 1
    ) AS pct_favoravel
  FROM legisladores l
  JOIN financiamentos_tse f ON f.legislador_id = l.id
  JOIN votos_individuais vi ON vi.legislador_id = l.id
  JOIN votacoes v ON v.id = vi.votacao_id
  JOIN proposicoes p ON p.id = v.proposicao_id
  WHERE p.temas_farol @> ARRAY[$1::text]
    AND f.setor_normalizado = $2
  GROUP BY l.id, l.nome_completo, l.partido_atual, f.setor_normalizado
  HAVING SUM(f.valor) > 10000
  ORDER BY total_recebido DESC
`;

Metodologia de Scoring Constitucional

Esta é a peça mais sensível e distintiva do Farol. O objetivo não é produzir um tribunal paralelo, mas um sistema de sinalização de risco que ative a atenção pública e de especialistas quando um projeto legislativo pode violar a Constituição Federal de 1988. O modelo é explicitamente auditável e sempre acompanhado de justificativa textual.

Princípio epistemológico: o score é um índice de risco constitucional, não uma decisão jurídica. Toda proposição com score ≥ 0.6 ("risco alto") é enfileirada para revisão por especialistas humanos antes de ser exibida publicamente com o flag de risco. A IA identifica; humanos validam.

Mapeamento CF/88 → Temas Legislativos

Os artigos da Constituição são mapeados para os mesmos temas usados para classificar proposições. Isso permite cruzar automaticamente: "este PL toca em matéria trabalhista → verificar Art. 7º".

Componentes do Score

⚖ Conflito com Cláusulas Pétreas

Máxima severidade. Cláusulas pétreas (Art. 60 §4) são imutáveis mesmo por PEC. Qualquer proposta que toque em separação de poderes, federação, voto direto ou direitos fundamentais recebe flag automático de revisão urgente.

Peso: 35% do score total
📋 Sobreposição com Direitos Sociais

Verifica se a proposição reduz direitos garantidos no Art. 6–11. Aplica o princípio da proibição de retrocesso social (não-retrogatividade), consolidado pela jurisprudência do STF.

Peso: 25% do score total
🔬 Precedente STF (ADIs e ADPFs)

Cruza a proposta com o banco de Ações Diretas de Inconstitucionalidade julgadas. Se temas similares já foram declarados inconstitucionais, o risco sobe automaticamente.

Peso: 25% do score total
📐 Conformidade Procedimental

Verifica se o tipo de proposição é compatível com a matéria regulada. Ex: alguns temas exigem PEC, não PL ordinário. Detecta tentativas de legislar via instrumento inadequado.

Peso: 15% do score total

Implementação do Analisador

TypeScript — análise constitucional com Claude
const CF88_SYSTEM_PROMPT = `
Você é um analista constitucional especializado na Constituição Federal
do Brasil de 1988 (CF/88). Sua função é avaliar proposições legislativas
e identificar potenciais conflitos constitucionais com base em:

1. ARTIGOS RELEVANTES: identifique quais artigos da CF/88 são tocados
2. CLÁUSULAS PÉTREAS: verifique se Art. 60 §4 é relevante
3. JURISPRUDÊNCIA STF: mencione ADIs ou ADPFs relacionadas se souber
4. PRINCÍPIO DE NÃO-RETROCESSO: aplique em direitos sociais (Art. 6-11)
5. LAICIDADE: verifique Art. 19 se matéria tiver cunho religioso

Responda APENAS em JSON com a seguinte estrutura:
{
  "score": número entre 0.0 (sem risco) e 1.0 (risco crítico),
  "nivel_risco": "baixo" | "medio" | "alto" | "critico",
  "artigos_relevantes": [{"artigo": "Art. X", "inciso": "...", "relevancia": "..."}],
  "conflito_clausula_petrea": boolean,
  "retrocesso_social": boolean,
  "justificativa": "texto em linguagem acessível ao cidadão (máx 200 palavras)",
  "justificativa_tecnica": "texto para juristas (máx 300 palavras)",
  "recomendacao_expert": boolean
}
`;

async function analisarConstitucionalidade(
  proposicao: Proposicao,
  precedentesSTF: PrecedenteSTF[]
): Promise<ScoreCF88> {
  const contexto = montarContexto(proposicao, precedentesSTF);

  const response = await anthropic.messages.create({
    model: 'claude-sonnet-4-20250514',
    max_tokens: 1500,
    system: CF88_SYSTEM_PROMPT,
    messages: [{
      role: 'user',
      content: `
PROPOSIÇÃO: ${proposicao.siglaTipo} ${proposicao.numero}/${proposicao.ano}
EMENTA: ${proposicao.ementa}
TEMAS: ${proposicao.temas_farol.join(', ')}
TEXTO PARCIAL: ${proposicao.ementa_detalhada}
PRECEDENTES STF RELEVANTES: ${JSON.stringify(precedentesSTF, null, 2)}
      `
    }]
  });

  const raw = response.content[0].text;
  const parsed: ScoreCF88Raw = JSON.parse(raw);

  return {
    ...parsed,
    proposicao_id: proposicao.id,
    revisado_por_expert: false, // sempre inicia como false
    modelo_versao: 'claude-sonnet-4-20250514',
    calculado_em: new Date().toISOString()
  };
}

// Workflow completo: score → validação → publicação
async function processarAnalise(proposicaoId: string) {
  const [proposicao, precedentes] = await Promise.all([
    db.findProposicao(proposicaoId),
    stf.buscarPrecedentes(proposicaoId)
  ]);

  const score = await analisarConstitucionalidade(proposicao, precedentes);
  await db.upsertScore(score);

  // Proposta de alto risco → fila de revisão humana
  if (score.score >= 0.6 || score.conflito_clausula_petrea) {
    await notificacoes.enviarParaRevisores(score);
    await queue.enqueue('revisao-expert', { scoreId: score.id });
  }
}

Governança e Sustentabilidade

Um projeto de infraestrutura cívica que monitora o poder tem uma equação de governança específica: precisa ser financeiramente independente do Estado, tecnicamente auditável, juridicamente defensável, e politicamente neutro em sua metodologia (não em seus achados).

Modelo Institucional

🏛

Forma Jurídica

OSCIP ou Associação

Organização da Sociedade Civil de Interesse Público (OSCIP) permite receber doações com benefício fiscal, assinar termos de parceria com governo sem ser controlada por ele, e ter estrutura de governança transparente por lei. A associação simples é mais fácil de criar, mas a OSCIP dá mais credibilidade a financiadores institucionais.

⚖

Conselho de Governança

Tripé de Accountability

Conselho Diretivo: 5 pessoas, mandato rotativo de 2 anos, incluindo pelo menos 1 jurista constitucionalista, 1 especialista em dados abertos, 1 representante de sociedade civil organizada.

Conselho Fiscal: 3 pessoas independentes, auditoria anual publicada.

Comitê Científico: acadêmicos voluntários que validam a metodologia de scoring.

💻

Código e Dados

Open Source Total

Todo o código é AGPL-3.0 — qualquer derivação também deve ser aberta. Os dados processados são publicados sob CC0 (domínio público). Os prompts de análise constitucional são versionados e publicados no repositório. Qualquer pessoa pode auditar como um score foi gerado.

🔍

Revisão Especializada

Camada Humana Obrigatória

Scores ≥ 0.6 nunca são publicados sem revisão humana. Uma rede de juristas voluntários (via parceria com OAB, clínicas jurídicas universitárias) valida os alertas antes de exibição pública. Isso protege o projeto de ações de deputados atingidos e mantém credibilidade.

Matriz de Riscos

Risco Probabilidade Impacto Mitigação
Ação judicial de deputado por score de risco Média Alto Toda análise ≥0.6 tem revisão jurídica antes de publicar. Disclaimers claros: é sinalização de risco, não sentença.

Modelo Financeiro

Fontes Priorizadas

Fundações filantrópicas
Ford Foundation, Open Society, Luminate, Knight Foundation têm linhas específicas para democracia digital e transparência. São fontes ideais: não têm agenda política brasileira e têm histórico de financiar projetos similares (Serenata de Amor, Brasil.IO).

Crowdfunding cívico
Subscritores mensais via Catarse ou Apoia.se. Independência + base ativa de apoiadores que são também embaixadores.

Parcerias com mídia
Agência Pública, The Intercept Brasil, Piauí: licença de uso dos dados processados em troca de financiamento ou visibilidade.

Fontes Excluídas (por protocolo)

Partidos políticos — conflito de interesse direto com o objeto monitorado.

Governo federal, estadual ou municipal — dependência que compromete independência editorial.

Empresas reguladas pelo Congresso — agronegócio, telecom, sistema financeiro, saúde privada. Doações destes setores são exatamente o que o sistema monitora.

Fontes anônimas — toda doação acima de R$500 é publicada no relatório anual.
